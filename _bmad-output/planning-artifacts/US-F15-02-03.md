# Story F15-02.03: Reverse Image Search activación (feature flag on)

## Story

**Como** analista de pricing de MT,
**Quiero** activar la búsqueda inversa de imágenes para candidatos con baja confianza calibrada,
**Para** que el pipeline rescate matches legítimos cuyas imágenes de catálogo no superan el umbral de embedding/VLM, detectando fotos reutilizadas por múltiples vendors.

---

## Acceptance Criteria

**AC#1 — Flag + provider routing**
**Dado** que el feature flag `reverse_image_search` está activo y `REVERSE_IMAGE_PROVIDER=tineye` o `google_lens_serpapi`
**Cuando** el pipeline obtiene `calibrated_confidence < 0.50` para un candidato tras embedding, OCR y VLM judge
**Entonces** se invoca `ReverseImageSearchPort.search(image_url=...)` antes del descarte definitivo

**AC#2 — Adapters reales**
**Dado** `REVERSE_IMAGE_PROVIDER=tineye`
**Cuando** se invoca `search()`
**Entonces** `TinEyeAdapter` llama a TinEye API con `TINEYE_API_KEY` via httpx y mapea respuesta a `ReverseImageSearchResult`

**Dado** `REVERSE_IMAGE_PROVIDER=google_lens_serpapi`
**Cuando** se invoca `search()`
**Entonces** `GoogleLensSerpApiAdapter` llama a SerpAPI con `SERPAPI_KEY` via httpx y mapea respuesta a `ReverseImageSearchResult`

**AC#3 — Daily limit**
**Dado** que se alcanza `REVERSE_IMAGE_DAILY_LIMIT` (default 200) llamadas en el día
**Cuando** se invoca `search()`
**Entonces** retorna `ReverseImageSearchResult` con `hits=()` y `provider="limit_reached"`, sin error, candidato continúa sin boost

**AC#4 — Whitelist boost**
**Dado** que algún hit tiene `domain` perteneciente a la whitelist de dominios canónicos del fabricante del SKU
**Cuando** se evalúa el resultado RIS
**Entonces** `calibrated_confidence` recibe boost `+0.15` (capped a 1.0) y se registra `method='embedding+ris'` en la decisión

**AC#5 — Persistencia**
**Dado** que se ejecutó RIS para un candidato
**Cuando** se persiste la decisión
**Entonces** `competitor_listings.reverse_image_hits JSONB`, `reverse_image_searched_at TIMESTAMPTZ`, `reverse_image_provider VARCHAR(32)` se actualizan via migration 071

**AC#6 — Noop cuando flag OFF**
**Dado** que el feature flag `reverse_image_search` está OFF
**Cuando** el pipeline procesa un candidato con baja confianza
**Entonces** `NoopRisAdapter.search()` retorna `ReverseImageSearchResult(provider='noop', hits=(), searched_at=...)` sin latencia adicional ni llamadas externas

**AC#7 — Flag togglable en caliente**
**Dado** que `flag_service.py` ya implementa toggle Redis + DB con TTL 60s
**Cuando** se cambia el flag `reverse_image_search` vía admin
**Entonces** el adapter activo cambia sin reinicio del worker (SLA 60s propagación)

    ```python
    REDIS_KEY = "mt:ris:daily_count:{date}"  # date = YYYY-MM-DD
    
    async def check_and_increment(redis: RedisLike, *, limit: int) -> bool:
        """Retorna True si se puede llamar (bajo el límite), False si se alcanzó."""
        today = datetime.now(tz=timezone.utc).strftime("%Y-%m-%d")
        key = REDIS_KEY.format(date=today)
        count = await redis.incr(key)
        if count == 1:
            await redis.expire(key, 86400)  # TTL 24h
        return count <= limit
    ```

## Dev Notes

### Contexto en el pipeline

El pipeline de comparación (ADR-012) tiene la siguiente secuencia de stages:
```
Embedding ANN → OCR → Calibración → VLM Judge → [RIS si confidence < 0.50] → Decisión final
```

RIS se posiciona **después** de VLM Judge como stage de rescate para candidatos con baja confianza. El objetivo no es maximizar coverage sino rescatar matches legítimos con fotos compartidas entre vendors.

### Adapters: TinEye API

TinEye JSON API v1.1:
- Endpoint: `GET https://api.tineye.com/rest/search/?image_url={url}&limit={n}&api_key={key}`
- Respuesta: `{"results": {"matches": [{"image_url": "...", "domain": "...", "score": 0.9}]}}`
- Autenticación: query param `api_key`
- Sin `domain` en hit → extraer con `urllib.parse.urlparse(url).netloc`
- Sin `score` → usar `0.0`
- Timeout: `httpx.AsyncClient(timeout=10.0)`

### Adapters: SerpAPI Google Lens

SerpAPI:
- Endpoint: `GET https://serpapi.com/search?engine=google_lens&url={image_url}&api_key={key}`
- Respuesta: `{"visual_matches": [{"link": "...", "source": "domain.com", "thumbnail": "..."}]}`
- Mapear `link` → `url`, `source` → `domain`, `thumbnail` → `thumbnail_url`, similarity = `0.0`
- Timeout: `httpx.AsyncClient(timeout=15.0)` (SerpAPI es más lento)
