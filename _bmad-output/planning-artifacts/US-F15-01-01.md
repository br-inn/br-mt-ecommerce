# US-F15-01-01 — Neo4j setup + validación residencia datos UAE

**Épica**: EP-F15-01 (KG Foundation) | **Sprint**: S10 | **SP**: 3 | **Prioridad**: P0 — BLOQUEANTE

## Contexto

Wave 1 de S10. Bloqueante para todas las stories F15-01-02/03/04.
Neo4j 5.20 ya tiene driver/adapter/factory en el codebase (S9). Solo falta
el servicio Docker y las variables de entorno. `app/core/config.py` ya define
`NEO4J_URI`, `NEO4J_USER`, `NEO4J_PASSWORD`, `NEO4J_DATABASE` etc.

## Descripción

Confirmar y completar la configuración de Neo4j 5.20 en el stack Docker local
de desarrollo. Crear documentación de residencia de datos UAE. Validar healthcheck.

## Criterios de Aceptación

- [ ] Servicio `neo4j` en `docker-compose.dev.yml`: imagen `neo4j:5.20`,
      puertos `17474:7474` (HTTP) y `17687:7687` (Bolt),
      `NEO4J_AUTH=neo4j/devpassword`, datos persistidos en volumen nombrado
- [ ] Variables en `.env.example`: `NEO4J_URI=bolt://localhost:17687`,
      `NEO4J_USER=neo4j`, `NEO4J_PASSWORD=devpassword`, `NEO4J_DATABASE=neo4j`,
      `GRAPHRAG_BACKEND=stub` (default seguro)
- [ ] Documento `docs/runbooks/neo4j-data-residency-uae.md` creado con:
      región de datos EU/AE, política no-Aura, estrategia backup Hetzner,
      nota de privacidad (ningún dato personal fuera de UAE)
- [ ] Script `scripts/healthcheck_neo4j.sh` que verifica Bolt con `cypher-shell`
      (o python driver) y retorna exit 0 si OK
- [ ] `docker compose up neo4j` levanta el servicio sin errores en dev local
- [ ] `GRAPHRAG_BACKEND=neo4j` conecta exitosamente al arrancar la app

## Notas Técnicas

- Imagen: `neo4j:5.20-community` (community edition suficiente para dev)
- Puerto interno Bolt: `neo4j:7687`, externo para dev local: `17687`
  (para no colisionar si hay otro Neo4j en el host)
- `NEO4J_AUTH=neo4j/devpassword` — solo para dev local, no para staging
- `app/core/config.py` líneas 131-138 ya tienen los defaults correctos:
  `NEO4J_URI: str = "bolt://neo4j:7687"` (nombre de servicio Docker)
- Para tests locales directo (fuera de Docker): `bolt://localhost:17687`
- No es necesario tocar `app/core/config.py` — ya está completo
