# Story F15-02.02: VLM Judge activación (claude-sonnet-4-6 audit-grade)

Status: done

## Story

Como analista de pricing o validador humano de MT,
Quiero que cada comparación de producto pase por un juez VLM basado en `claude-sonnet-4-6` que genere un veredicto estructurado con razonamiento en lenguaje natural,
Para que las decisiones de matching sean auditables y los validadores humanos comprendan el "por qué" antes del score numérico (anti-anchor bias).

## Acceptance Criteria

1. **[VLM call]** Dado que un candidato superó las etapas previas y `VLM_JUDGE_ENABLED=true`, cuando `ClaudeVlmJudgeAdapter` invoca `claude-sonnet-4-6` vía Anthropic SDK con imágenes SKU master y candidato, entonces el prompt solicita respuesta JSON exacta: `{"verdict":"match|reject|uncertain","confidence":0.0-1.0,"reasoning":"<1-3 frases>","deal_breakers_triggered":["..."],"image_regions":[{"side":"sku|candidate","description":"..."}]}` sin texto fuera del JSON.

2. **[Fallback JSON]** Respuesta validada con Pydantic (`ClaudeJudgeResponse`); si JSON inválido → veredicto `uncertain`, `confidence=0.0`, log WARNING.

3. **[Persistencia]** Resultado persistido en nuevas columnas de `match_decisions`: `judge_verdict VARCHAR(16)`, `judge_confidence NUMERIC(4,3)`, `judge_rationale TEXT`, `judge_image_regions JSONB`, `deal_breakers_triggered TEXT[]`, `judge_model_version VARCHAR(64)`, `judge_at TIMESTAMPTZ`.

4. **[Routing humano]** Si `judge_verdict='uncertain'` y `confidence < 0.50` → enrutamiento automático a `human_review_queue` con `reason='vlm_uncertain'`.

5. **[UI anti-anchor]** UI de validación humana renderiza `judge_rationale` ANTES del score numérico.

6. **[RBAC]** `judge_rationale` e `image_regions` restringidos a roles `admin`, `gerente`, `validador`; rol `viewer` recibe `null` en ambos.

7. **[Noop]** Si `VLM_JUDGE_ENABLED=false` o `ANTHROPIC_API_KEY` vacío → `NoopVlmJudgeAdapter` retorna `uncertain/0.0` sin llamadas externas.

8. **[Rate limit]** 1 req/s sostenido vía `aiolimiter.AsyncLimiter`, singleton de módulo compartido entre workers Celery.

9. **[Tests]** Unitarios cubren: JSON válido → veredicto correcto; JSON inválido → fallback uncertain; uncertain+confidence<0.50 → enqueue; flag OFF → noop.

### Anthropic SDK async

```python
import anthropic, json
from aiolimiter import AsyncLimiter

_VLM_RATE_LIMITER = AsyncLimiter(rate=1, time_period=1.0)  # singleton módulo

class ClaudeVlmJudgeAdapter(VlmJudgePort):
    def __init__(self, api_key: str) -> None:
        self._client = anthropic.AsyncAnthropic(api_key=api_key)

    async def judge(self, *, product_sku, candidate_image_url, product_image_url, context) -> VlmJudgeVerdict:
        async with _VLM_RATE_LIMITER:
            response = await self._client.messages.create(
                model="claude-sonnet-4-6",
                max_tokens=512,
                system=_SYSTEM_PROMPT,
                messages=[{"role": "user", "content": [
                    {"type": "image", "source": {"type": "url", "url": product_image_url}},
                    {"type": "image", "source": {"type": "url", "url": candidate_image_url}},
                    {"type": "text", "text": f"Context: {json.dumps(context)}\nRespond ONLY with valid JSON."}
                ]}]
            )
        try:
            parsed = ClaudeJudgeResponse.model_validate_json(response.content[0].text)
        except Exception:
            logger.warning("comparator.vlm_judge: invalid json response for sku=%s", product_sku)
            return VlmJudgeVerdict(decision="uncertain", confidence=0.0, rationale="", deal_breakers_triggered=())
        return VlmJudgeVerdict(
            decision=parsed.verdict,
            confidence=parsed.confidence,
            rationale=parsed.reasoning,
            deal_breakers_triggered=tuple(parsed.deal_breakers_triggered),
            image_regions=tuple(parsed.image_regions),
        )
```
