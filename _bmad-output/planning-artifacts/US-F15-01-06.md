# US-F15-01-06 — Dashboard monitoreo KG + integridad nightly (STRETCH)

**Épica**: EP-F15-01 (KG Foundation) | **Sprint**: S10 | **SP**: 5 | **Prioridad**: P2 — STRETCH

## Contexto

Wave 5 de S10 (stretch, paralelo con US-F15-01-05). Depende de US-F15-01-03 y US-F15-01-04.
Operability: sin visibilidad del KG es difícil detectar problemas de CDC o datos corruptos.

## Descripción

Endpoint `GET /graphrag/metrics` con estadísticas del KG. Celery Beat task
`kg_integrity_check` (02:00 UTC). Tabla `kg_integrity_results`. Alerta si hay
nodos huérfanos o lag CDC excesivo.

## Criterios de Aceptación

- [ ] `GET /api/v1/graphrag/metrics` retorna:
      ```json
      {
        "node_count": 12450,
        "edge_count": 8920,
        "orphan_nodes": 3,
        "cdc_lag_seconds": 1.2,
        "last_sync": "2026-05-12T10:30:00Z",
        "backend": "neo4j"
      }
      ```
      HTTP 503 con `{"error": "cdc_lag_too_high"}` si `cdc_lag_seconds > 300`
- [ ] Migración Alembic (mig 086) tabla `kg_integrity_results`:
      `id`, `checked_at`, `node_count`, `edge_count`, `orphan_nodes`,
      `cdc_lag_seconds`, `status (ok/warning/error)`, `details (jsonb)`
- [ ] Celery Beat task `mt.graphrag.kg_integrity_check` registrada en
      `job_definitions` (o equivalente en scheduler): schedule `0 2 * * *` (02:00 UTC)
      - Ejecuta queries Neo4j para métricas
      - Inserta en `kg_integrity_results`
      - Si `orphan_nodes > 10` o `cdc_lag > 300` → log ERROR estructurado
- [ ] Endpoint protegido con autenticación existente (JWT o API key según patrón)
- [ ] Tests: endpoint retorna 200 en stub mode, 503 cuando lag simulado > 300s

## Notas Técnicas

- `node_count`: `MATCH (n) RETURN count(n)`
- `edge_count`: `MATCH ()-[r]->() RETURN count(r)`
- `orphan_nodes`: `MATCH (n) WHERE NOT (n)--() RETURN count(n)`
- `cdc_lag_seconds`: diferencia entre `NOW()` y último `cdc_events.created_at` procesado
  (consulta Postgres, no Neo4j)
- En `GRAPHRAG_BACKEND=stub` modo: `node_count=0`, `edge_count=0`, `orphan_nodes=0`,
  `cdc_lag_seconds=0.0` — no 503
- Scheduler: ver `app/workers/tasks/` pattern existente para jobs periódicos (Beat)
