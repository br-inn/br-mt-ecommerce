# US-F15-01-05 — product_equivalences + ingestión fichas técnicas PDF (STRETCH)

**Épica**: EP-F15-01 (KG Foundation) | **Sprint**: S10 | **SP**: 8 | **Prioridad**: P2 — STRETCH

## Contexto

Wave 5 de S10 (stretch, paralelo con US-F15-01-06). Depende de US-F15-01-04.
Extiende el KG con relaciones de equivalencia entre productos MT — crucial para
el comparador en Fase 2. pdfplumber ya está en el codebase (S6).

## Descripción

Crear tabla `product_equivalences` + Celery task `ingest_equivalences_from_pdf`
que extrae equivalencias de fichas técnicas PDF y las sincroniza al KG de Neo4j
como aristas `EQUIVALENT_TO`.

## Criterios de Aceptación

- [ ] Migración Alembic (mig 085) tabla `product_equivalences`:
      `id (UUID PK)`, `product_id_a (FK products)`, `product_id_b (FK products)`,
      `confidence (float, 0-1)`, `source (str)`, `extracted_at (timestamp)`,
      `synced_to_kg (bool, default false)`
- [ ] Celery task `mt.graphrag.ingest_equivalences_from_pdf` en
      `app/workers/tasks/graphrag.py`:
      - Acepta `pdf_path: str` (ruta en Supabase Storage o local)
      - Extrae pares de equivalencia con pdfplumber + regex
      - Inserta en `product_equivalences`
      - Sincroniza a Neo4j como `EQUIVALENT_TO` edges con
        `confidence` y `source` como propiedades del edge
      - Idempotente: MERGE, no INSERT
- [ ] Si no hay PDFs en bucket → funciona con fixtures sintéticos
      (la task acepta `use_fixture: bool = False`)
- [ ] Tests con PDF fixture en `tests/fixtures/sample_equivalences.pdf`
      (PDF de 1 página con tabla de equivalencias sintética)

## Notas Técnicas

- Migración 085 — verificar que 084 (competitor_fetch_errors) fue creada en US-F15-02-01
- pdfplumber: `pip install pdfplumber` ya en pyproject.toml (S6)
- Regex para equivalencias: `r"([A-Z0-9\-]+)\s+(?:equiv\.?|=)\s+([A-Z0-9\-]+)"`
- Edge Cypher: `MERGE (a:Product {pk:$sku_a})-[r:EQUIVALENT_TO {source:$source}]->(b:Product {pk:$sku_b}) SET r.confidence=$confidence`
- Supabase Storage bucket `product-images` ya existe — usar mismo para PDFs o
  bucket separado `product-datasheets`
