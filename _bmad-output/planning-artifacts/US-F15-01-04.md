# US-F15-01-04 — Activar Neo4jGraphRepository + fix W-2/W-3

**Épica**: EP-F15-01 (KG Foundation) | **Sprint**: S10 | **SP**: 5 | **Prioridad**: P0

## Contexto

Wave 4 de S10. Depende de US-F15-01-01, US-F15-01-02, US-F15-01-03.
`Neo4jGraphRepository` en `app/services/comparator/graph_repository.py` actualmente
lanza `NotImplementedError` en todos sus métodos. Esta story implementa las queries
Cypher reales usando el `GraphStorePort` ya operativo.

Warningos conocidos del POC S9 que deben solucionarse:
- **W-2**: `health_check()` retorna `healthy: False` — must return `True` cuando Neo4j responde
- **W-3**: Tests marcados `@pytest.mark.neo4j_real` no corren en CI (falta marker config)

## Descripción

Implementar `get_product_neighbors` y `get_competitor_context` con Cypher real en
`Neo4jGraphRepository`. Fix `health_check()`. Configurar pytest marker `neo4j_real`.
`GET /graphrag/health` debe responder 200 con `healthy: true`.

## Criterios de Aceptación

- [ ] `Neo4jGraphRepository.get_product_neighbors` implementado:
      - Cypher: `MATCH (p:Product {primary_key: $sku})-[r]->(n) RETURN ...`
      - Filtra por `relationship_type` si se pasa
      - Retorna lista de dicts: `{node_type, primary_key, properties, relationship}`
      - Si Neo4j no disponible → retorna lista vacía (no lanza excepción)
- [ ] `Neo4jGraphRepository.get_competitor_context` implementado:
      - Cypher: `MATCH (c:CompetitorListing {primary_key: $id})<-[r]-(p:Product) ...`
      - Retorna dict: `{competitor_listing_id, product_matches, supplier_hints, graph_confidence}`
      - `graph_confidence > 0.0` cuando hay matches reales
- [ ] Fix W-2: `Neo4jGraphRepository.health_check()` retorna `{"healthy": True, ...}`
      cuando `graph_store.health_check()` retorna healthy, `False` si no
- [ ] Fix W-3: `pytest.ini` o `pyproject.toml` agrega marker
      `neo4j_real: marks tests requiring real Neo4j instance`
- [ ] `GET /api/v1/graphrag/health` retorna 200 con `{"healthy": true, "backend": "neo4j_graph_repository"}`
      cuando `GRAPHRAG_BACKEND=neo4j` y Neo4j operativo
- [ ] Tests `@pytest.mark.neo4j_real` en
      `tests/integration/services/comparator/test_neo4j_graph_repository.py`:
      neighbors de producto real, context de competitor real, health check

## Notas Técnicas

- `Neo4jGraphRepository` recibe `graph_store: GraphStorePort` vía DI
- Para queries directas Cypher (no MERGE), usar `graph_store.query_neighbors()`
  o ejecutar Cypher crudo via el driver (si `GraphStorePort` no expone query raw)
- Si `GraphStorePort` no tiene método `query_raw`, extender la interfaz o
  usar el driver directamente via `Neo4jGraphStore._session()` (internal)
- Alternativa limpia: agregar `query_neighbors_by_label(label, pk)` a `GraphStorePort`
- `graph_confidence` = `len(product_matches) / (len(product_matches) + 1)` como proxy
- El endpoint `/graphrag/health` ya existe en `app/api/routes/graphrag.py` —
  solo necesita delegar al `graph_repository.health_check()`
