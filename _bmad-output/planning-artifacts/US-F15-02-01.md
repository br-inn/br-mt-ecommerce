# US-F15-02-01 — Amazon SP API fetcher real (competitor pricing)

**Épica**: EP-F15-02 (Pipeline Activation) | **Sprint**: S10 | **SP**: 8 | **Prioridad**: P1

## Contexto

Wave 3-B de S10 (paralelo con US-F15-01-03). Independiente del KG.
El codebase ya tiene `AmazonSPApiAdapter` en `channel_mirror` para PUBLICAR
precios. Esta story crea un adapter SEPARADO en `comparator/` para OBTENER
precios de competidores desde Amazon UAE.

## Descripción

Implementar `AmazonSPApiFetcherAdapter` que obtiene precios de competidores
en Amazon UAE usando la SP-API real con LWA OAuth2. Fallback a stub cuando
`MT_LIVE_NETWORK != true` o credenciales no disponibles. Rate limit 2 req/s.

## Criterios de Aceptación

- [ ] Clase `AmazonSPApiFetcherAdapter` en
      `app/services/comparator/fetchers/amazon_sp_fetcher.py`:
      - LWA OAuth2: endpoint `https://api.amazon.com/auth/o2/token`,
        token cache TTL 3500s en Redis
      - Endpoint API: `https://sellingpartnerapi-eu.amazon.com`
      - Marketplace: `A2VIGQ35RCS4UG` (Amazon.ae)
      - Rate limit: 2 req/s via `aiolimiter.AsyncLimiter`
      - Retry: `tenacity` async, 3 intentos, wait_exponential
      - Fallback automático a `AmazonSPFetcherStub` si
        `settings.MT_LIVE_NETWORK != "true"` o faltan credenciales

## Notas Técnicas

- **No copiar** `channel_mirror/adapters/amazon_sp_api.py` — ese adapter es
  para publicar (push), este es para fetching (pull). APIs y auth flow difieren.
- LWA token refresh: POST a LWA URL con `grant_type=refresh_token` + creds
- Token cache en Redis: key `sp_api:lwa_token:{seller_id}`, TTL 3500s
- Separar la lógica de fetch del registro de errores (tabla vs excepción)
- `aiolimiter`: `AsyncLimiter(rate=2, time_period=1.0)` — async context manager
- Usar `httpx.AsyncClient` (no `requests`) para consistencia con el stack async
- Si `GRAPHRAG_BACKEND=stub`, el fetcher puede correr en modo live de todos modos
  (son concerns independientes)
