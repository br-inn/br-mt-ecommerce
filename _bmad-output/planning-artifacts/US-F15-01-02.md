# US-F15-01-02 — Schema KG + seed Compatibilidad Materiales (657 filas)

**Épica**: EP-F15-01 (KG Foundation) | **Sprint**: S10 | **SP**: 5 | **Prioridad**: P0

## Contexto

Wave 2 de S10. Depende de US-F15-01-01 (Neo4j operativo).
Establece el schema del Knowledge Graph y popula datos de compatibilidad de
materiales que serán la fuente de verdad para el comparador en Fase 1.5.

## Descripción

Crear script idempotente `scripts/seed_kg_materials.py` que aplica constraints
de unicidad en Neo4j 5 (4 node types) y hace MERGE de los datos de compatibilidad
de materiales. CSV sintético generado para bootstrap; se refina con datos reales en S11.

## Criterios de Aceptación

- [ ] Script `scripts/seed_kg_materials.py` idempotente (MERGE, no INSERT):
      - Crea constraints de unicidad `(Label, primary_key)` en 4 node types:
        `Material`, `Standard`, `Supplier`, `Product`
      - Lee CSV desde `scripts/seed_data/kg_materials_seed.csv`
      - Ejecuta MERGE para cada par de materiales compatibles
      - Soporta `--dry-run` flag
      - Exit 0 si OK, exit 1 con mensaje claro si falla
- [ ] CSV `scripts/seed_data/kg_materials_seed.csv` con 657+ filas, columnas:
      `material_a,material_b,pressure_bar,temperature_range,standard,confidence`
      (datos sintéticos representativos de industria de válvulas/tuberías)
- [ ] Constraints en Neo4j aplicadas correctamente (verificable con `SHOW CONSTRAINTS`)
- [ ] Test `tests/integration/services/graphrag/test_kg_seed.py` con
      `@pytest.mark.neo4j_real` que verifica: constraints creadas, node count > 500,
      idempotencia (doble ejecución no duplica nodos)

## Notas Técnicas

- Usar `neo4j.GraphDatabase.driver` (sync) — mismo patrón que `neo4j_real.py`
- Node types prioritarios: `Material` (material de construcción), `Standard`
  (norma ISO/ASTM/DIN), `Supplier` (proveedor), `Product` (referencia MT)
- Edge types: `COMPATIBLE_WITH`, `CERTIFIED_BY`, `SUPPLIED_BY`, `MATCHES`
- CSV generado sintéticamente con ~150 materiales × combinaciones comunes
  (acero inox 316L, PTFE, Viton, EPDM, etc.) en rangos industriales típicos
- Script debe conectar a `settings.NEO4J_URI` — respetar config existente
- No crear migraciones Alembic — Neo4j no usa Alembic
